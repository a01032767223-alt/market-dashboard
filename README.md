# 국내증시 관제탑

KOSPI·KOSDAQ 지수, 5단계 매수전략, 섹터 바스켓 스코어링, 신호 변화 추적을 한 화면에서 보는 대시보드입니다.
GitHub Actions가 장중 5분마다 한국투자증권(KIS) API에서 데이터를 받아 `data.json`으로 저장하고,
GitHub Pages가 그 파일을 읽어 화면을 그립니다.

## 파일 구성

| 파일 | 역할 |
|------|------|
| `index.html` | 대시보드 화면 (GitHub Pages가 서빙) |
| `fetch_data.py` | KIS·글로벌 지표 수집 → `data.json` 생성 |
| `.github/workflows/update.yml` | 5분마다 수집 실행 (평일 장중) |
| `data.json` | 자동 생성됨. 직접 만들 필요 없음 |

## 최초 설정

1. **시크릿 등록** — Settings → Secrets and variables → Actions → New repository secret

   | 이름 | 값 |
   |------|-----|
   | `KIS_APP_KEY` | 한국투자증권 앱키 |
   | `KIS_APP_SECRET` | 앱시크릿 |

   이름 철자가 다르면 실행이 실패합니다.

2. **Pages 활성화** — Settings → Pages → Source를 `main` 브랜치 루트(`/`)로 지정

3. **첫 실행** — Actions 탭 → "KIS 데이터 갱신" → Run workflow
   1분 뒤 저장소 루트에 `data.json`이 생기면 성공입니다.

4. **확인** — Pages 주소를 열어 상단에 "갱신 HH:MM 기준"이 뜨면 실데이터입니다.
   이 표시가 없으면 아직 데모 데이터입니다.

## 운영 메모

- **자동 실행 시간**: 평일 09:00~17:00(KST) 5분 간격. 그 외 시간에는 수동 실행만 가능합니다.
- **신호 변화 감시**: 히스토리가 쌓여야 표시됩니다. 6갱신(약 30분) 후 6갱신 변화가,
  12갱신(약 1시간) 후 12갱신 변화가 나타납니다. 그 전에는 "히스토리를 모으는 중" 안내가 뜹니다.
- **고점(ATH) 자동 갱신**: 신고가가 나오면 자동으로 상향됩니다. 수동 수정이 필요 없습니다.
  단 KOSDAQ 기준값은 2026년 사이클 고점(1,214)이며 역대 최고가(2000년 2,800선대)가 아닙니다.
- **일부 지표 실패**: 화면 상단에 경고가 뜨고 해당 값은 직전 데이터로 유지됩니다. 전체가 멈추지 않습니다.
- **60일 무활동**: GitHub는 60일간 활동이 없으면 스케줄 워크플로를 자동 중단합니다.
  이 저장소는 갱신마다 커밋하므로 대체로 유지되지만, 멈추면 메일이 오고 버튼 한 번으로 재개됩니다.

## 로컬에서 쓰기 (선택)

PC에서 즉석 조회하려면 별도의 `kis_proxy.py`를 로컬에만 두고 실행합니다.
이 파일은 앱키를 코드에 직접 넣는 방식이라 **저장소에 올리지 마세요** (`.gitignore`에 등록돼 있습니다).
로컬 프록시는 히스토리를 만들지 않으므로 신호 변화 감시 기능은 Pages 쪽에서만 동작합니다.

## 투자 유의

이 대시보드는 지표를 모아 보기 위한 참고 도구입니다. AI 점수는 모멘텀·상대강도·수급·추세를
정해진 가중치로 계산한 규칙 기반 점수일 뿐 미래 수익률을 예측하지 않으며,
특정 종목·섹터의 매매를 권유하지 않습니다. 모든 투자 판단과 책임은 본인에게 있습니다.
